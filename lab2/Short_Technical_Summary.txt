This project performs binary pet segmentation on Oxford-IIIT Pet using UNet and ResNet34_UNet. Final models were trained and inferenced with:
- python src/train.py --model unet --seed 42
- python src/train.py --model resnet34_unet --seed 3407
- python src/inference.py --model unet --test_txt dataset/test_unet.txt --tta_scales 0.9,1.0,1.1
- python src/inference.py --model resnet34_unet --test_txt dataset/test_res_unet.txt --tta_scales 0.9,1.0,1.1

Implementation highlights:
- UNet follows the original design (no BatchNorm, no conv padding, crop-based skip).
- ResNet34_UNet uses a ResNet34 encoder, U-Net decoder, skip concatenation, and CBAM.
- Minor adjustments were made in ResNet34_UNet to resolve channel mismatches in the provided diagram while preserving the overall architecture.

Data preprocessing: 
- Images resized to a fixed resolution.
- UNet uses reflect padding and output center-cropping to handle valid convolutions.
- Data augmentation includes random scaling and rotation.

Training strategy:
- AdamW optimizer with cosine annealing.
- Loss: BCEDiceLoss for better overlap learning.
- Early stopping applied.
- Best threshold selected via validation search instead of fixed 0.5.

Key observations:
- Proper spatial alignment is critical for UNet performance.
- Threshold tuning improves Dice score.
- Test-time augmentation (multi-scale + flip) provides small gains.

Comparison:
- UNet is sensitive to geometric handling due to valid convolutions but achieves stable results after alignment fixes.
- ResNet34_UNet is easier to train and more robust due to deeper encoder features.
- Overall, ResNet34_UNet shows better generalization, while UNet requires more careful preprocessing.

Hardware: NVIDIA RTX 5090 GPU, Python 3.10.

Declaration: ChatGPT assisted with parts of implementation (BCEDiceLoss, threshold search, padding/cropping, CBAM, validation visualization) and summary writing.
